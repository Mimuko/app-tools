# consultation_type ごとの STATE0 表示内容と妥当性

## 一覧

| 相談タイプ | 表示フィールド | request_intent の選択肢 | 妥当性メモ |
|------------|----------------|--------------------------|------------|
| **不具合の可能性がある** | 相談タイプ・今欲しい対応・**今回の変更・対応内容**・対象・操作手順・環境・今の状態・理想の状態・参考 | **すべて**（表示変更／機能追加・変更／不具合修正／以前の仕様に戻したい／まだ整理できていない） | 不具合疑いでは「不具合を修正したい」「以前の仕様に戻したい」が自然。表示変更・機能変更もあり得るため全選択肢で問題なし。 |
| **仕様かどうか確認したい** | 相談タイプ・今欲しい対応・対象・操作手順・環境・今の状態・理想の状態・参考（**今回の変更・対応内容は非表示**） | （質問ごと非表示） | 「どんな変更をしたいか」より「仕様かどうか確認したい」が主目的のため、request_intent を出さない設計で妥当。 |
| **変更・改善を検討したい** | 相談タイプ・今欲しい対応・**今回の変更・対応内容**・対象・今の状態・理想の状態・環境・参考。<br>**「修正してほしい」選択時は操作手順を非表示**（STATE0_change_consult_fix）。 | request_intent: **表示変更／機能追加・変更／まだ整理できていない** のみ。<br>ask_type: **仕様を確認したい／修正の見積がほしい／修正してほしい** のみ。 | 変更・改善＋修正依頼の場合は操作手順不要のため非表示。妥当。 |
| **まだ整理できていない** | 相談タイプ・今欲しい対応・**今回の変更・対応内容**・対象・操作手順・環境・今の状態・理想の状態・参考 | **すべて** | 意図が固まっていないため、全選択肢を表示して書けるところから選んでもらう設計で妥当。 |

## 参照 CSV

- **表示フィールド**: `FIELD_SETS.csv` の `STATE0_<consultation_type>`
- **request_intent の選択肢絞り込み**: `FIELD_OPTIONS_BY_CONSULTATION.csv`（未定義の consultation_type は全選択肢）
